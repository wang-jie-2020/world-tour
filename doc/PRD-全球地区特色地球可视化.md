# PRD：全球地区特色地球可视化（需求重梳理版）

## 1. 文档信息
- 版本：v1.2
- 日期：2026-06-01
- 状态：基线版本（需求重梳理后）
- 关联实现：`src/app.ts`、`src/data.ts`、`src/style.css`
- 视觉参考：`doc/video-frames/frame_001.jpg` ~ `frame_017.jpg`

## 2. 重梳理结论
本版本解决以下问题：
1. 叙事主题不再强绑定到固定 UI 形态。
2. 叙事点不再写死“每区 3-5”，改为配置驱动。
3. 验收必须覆盖运行态与视觉，不再仅依赖构建通过。
4. 增强项不得替代核心目标。

## 3. 项目目标
在可旋转地球基础上，实现“地区特色叙事”的可视化体验：
1. 用户可快速识别区域差异。
2. 用户可切换并感知不同叙事主题（如花朵、童话、景点）。
3. 用户可与叙事节点交互并获得可读信息反馈。
4. 视觉质感向关键帧靠拢，且移动端可用。

## 4. 非目标
- 不做天气/气候模拟系统。
- 不将昼夜系统作为核心叙事。
- 不追求 GIS 级行政边界精度。
- 不把特定控件样式（如 chips）当作产品目标本身。

## 5. 视觉对齐原则
1. 关键帧仅用于对齐构图、密度、层次、色彩氛围、UI 侵入度。
2. 关键帧中出现的非目标元素不自动纳入产品范围。
3. 对齐基准镜头为 `KF01 / KF08 / KF13 / KF17`。

## 6. 功能需求（能力导向）
### 6.1 地球与相机能力
- 支持鼠标/触控旋转、缩放、阻尼。
- 支持自动自转开关。
- 区域或叙事节点聚焦时，相机平滑过渡。

### 6.2 区域表达能力（最高优先级）
- 覆盖：欧洲、东亚、南亚、东南亚、北美、南美、非洲、大洋洲。
- 每个区域必须具备：
  - 区域底色/纹理差异；
  - 可见边界（线或粒子）；
  - 选中高亮态（halo 或等价效果）。

### 6.3 主题系统能力（配置驱动）
- 系统支持主题注册、启用、禁用、叠加。
- 默认主题集可为：`flowers`、`fairy`、`destinations`。
- 主题集合可配置，可增删替换，不要求改动 UI 结构。

### 6.4 叙事节点能力（配置驱动，替代固定点位规则）
- 叙事节点按“地区 x 主题”配置，不做统一固定数量约束。
- 支持按场景配置节点数量：`0..N`（可为空、可密集）。
- 节点字段至少包含：标识、所属地区、所属主题、坐标、展示文案。
- 支持点击节点显示信息卡（名称/标签/描述等可配置字段）。
- 信息卡在桌面与移动端可读，不遮挡核心交互区域。

### 6.5 叙事联动能力
- 点击节点后支持镜头聚焦联动。
- 节点可选支持连线、光束、粒子等主题化表现。
- 联动逻辑应由配置控制开关与强度，不写死到单一主题。

### 6.6 控制能力（不绑定具体控件形态）
- 必须提供以下控制能力：
  - 区域切换；
  - 主题切换；
  - 强度/密度调节；
  - 动画开关（自转、导览）。
- UI 可采用 chips、下拉、面板或混合形态；PRD 仅约束能力，不强制样式。
- 提供展示态/编辑态切换，默认展示态降低工具感。

### 6.7 A/B 评估能力
- 支持 `Target/Baseline` 切换。
- 支持关键帧镜头循环，便于截图对比。

## 7. 动效量化要求
- 区域切换过渡：200-500ms。
- 主题开关：渐入渐出，无明显闪烁与突变。
- 节点聚焦：镜头路径连续、无跳变。

## 8. 性能与兼容
- 桌面端：主流设备交互流畅，视觉连续。
- 移动端：可完成旋转、缩放、切换、点击四类核心交互。
- 低性能设备：自动降级粒子预算与后处理强度。

## 9. 验收标准（DoD）
### 9.1 功能验收（运行态）
1. 用户可在 3 秒内识别当前聚焦区域。
2. 主题切换后视觉差异明确，且区域仍可识别。
3. 存在可交互叙事节点，并可弹出信息卡。
4. 桌面与移动端均可完成旋转、缩放、切换、点击。

### 9.2 配置化验收（新增）
1. 叙事节点数量可通过配置调整，无需改 UI 结构代码。
2. 新增或替换主题时，可通过配置接入并完成开关展示。
3. 至少验证两组不同节点规模配置（稀疏/密集）均可正常运行。

### 9.3 技术验收
1. `npm run build` 通过（仅技术门禁，不替代功能验收）。
2. 运行态控制台无阻断性错误。
3. 无严重视觉缺陷（闪烁、穿帮、交互中断）。

### 9.4 视觉验收
按 `doc/FRAME-EVAL-RUBRIC.md` 对 `KF01/KF08/KF13/KF17` 评分：
- 单帧 >= 64/80；
- 四帧均分 >= 68/80；
- 无 blocking 级缺陷。

## 10. 里程碑与退出条件
- M1 区域可识别性：底色/边界/选中态全部可见并可切换。
- M2 主题系统解耦：主题集合可配置，主题切换可辨识。
- M3 叙事节点闭环：节点配置、信息卡、镜头联动完成。
- M4 视觉与性能收敛：关键帧评分与运行态验收达标。

## 11. 范围管理
### 11.1 核心范围
- 仅限第 6 节与第 9 节定义项。

### 11.2 增强范围
- 核心范围达标后方可进入增强项。
- 增强项单独记录，不得用于声明核心目标完成。

## 12. 风险与应对
- 风险：叙事节点配置过密导致视觉噪声。
  - 应对：提供密度上限与主题权重控制。
- 风险：主题扩展破坏一致性。
  - 应对：主题注册规范化，接入时走关键帧与运行态验收。
- 风险：仅通过构建门禁但体验不达标。
  - 应对：发布前强制要求运行态验收与关键帧评分。
